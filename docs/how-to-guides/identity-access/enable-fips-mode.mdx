## What you'll accomplish

You'll verify that FIPS 140-2 Strict Mode is active in your UDS Core Keycloak deployment and understand its constraints.

> [!NOTE]
> **FIPS 140-2 Strict Mode is always enabled in UDS Core.** The UDS Identity Config init container automatically copies the required Bouncy Castle FIPS JAR files into Keycloak's providers directory. No additional configuration is needed.

## Prerequisites

- UDS Core deployed
- [UDS CLI](https://github.com/defenseunicorns/uds-cli/releases) installed

## Before you begin

FIPS mode changes how Keycloak handles cryptography and passwords:

- `argon2` (the upstream Keycloak default) is not FIPS-approved and is unavailable — UDS Core uses `pbkdf2-sha256` by default
- All passwords must be at least 14 characters
- Only FIPS-approved algorithms are available for signing, encryption, and hashing

## Verification

Temporarily enable debug mode to confirm FIPS is active:

```yaml title="uds-bundle.yaml"
- path: debugMode
  value: true
```

Deploy the bundle, then check the Keycloak startup logs:

```bash
uds zarf tools kubectl logs -n keycloak -l app.kubernetes.io/name=keycloak --tail=100 | grep BCFIPS
```

Look for:

```
KC(BCFIPS version 2.0 Approved Mode, FIPS-JVM: disabled)
```

`BCFIPS version 2.0 Approved Mode` confirms Keycloak is running in FIPS Strict Mode. `FIPS-JVM: disabled` is expected unless the underlying host OS is also running a FIPS-enabled kernel. Disable `debugMode` once you've confirmed FIPS is active.

> [!NOTE]
> For details on FIPS constraints and limitations in Keycloak, see the [Keycloak FIPS 140-2 support docs](https://www.keycloak.org/server/fips).

## Migrating from non-FIPS deployments

If you are upgrading from an older deployment where FIPS was not enabled and user accounts already exist, you need to migrate password hashing before FIPS can activate cleanly.

1. Before upgrading, connect to the Keycloak admin console (for example, via `uds zarf connect keycloak`)
2. Switch to the **master** realm → **Authentication** → **Policies** → **Password Policy** → add `pbkdf2-sha512` as the hashing algorithm
3. Reset the admin user's password to a 14+ character value with **Temporary** set to **Off**
4. Repeat for any other local users whose passwords were hashed with `argon2`
5. Upgrade to the latest UDS Core version

## Troubleshooting

### Problem: Keycloak admin console is inaccessible after upgrade

**Symptoms:** Cannot log in to the Keycloak admin console after upgrading to a UDS Core version with FIPS enabled. Login fails with a password error.

**Solution:** This happens when the admin password was hashed with `argon2` (the upstream Keycloak default before FIPS) or is shorter than 14 characters — FIPS mode rejects both. Follow the [migration steps](#migrating-from-non-fips-deployments) on a pre-upgrade deployment to resolve this before upgrading.
