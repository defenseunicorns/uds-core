UDS Core 1.3 introduces opt-in support for public Keycloak clients (PKCE-enforced flows beyond device flow), automatic gateway pod cycling when Istio `gatewayTopology.proxyProtocol` changes, and an Identity Config workflow for disabling inactive non-admin users. This release also picks up the Prometheus 3.11.2 stored XSS fix (CVE-2026-40179) and rolls in routine dependency updates for Istio, Pepr, and the kube-prometheus-stack chart.

### Notable features

- **Opt-in public Keycloak clients:** adds an `ALLOW_PUBLIC_CLIENTS` operator config flag (default `false`) that gates admission of UDS Package SSO clients with `publicClient: true` for flows beyond device flow. Public clients still require PKCE S256, and the package validator now enforces it. See [ADR 0010](https://github.com/defenseunicorns/uds-core/blob/main/adrs/0010-allow-opt-in-public-clients.md) and [#2598](https://github.com/defenseunicorns/uds-core/pull/2598)
- **Gateway cycling on `proxyProtocol` changes:** the Pepr operator now restarts gateway pods when `meshConfig.defaultConfig.gatewayTopology.proxyProtocol` is toggled. `proxyProtocol` is read at pod startup and is not pushed via xDS, so toggling it via a bundle override previously left gateway pods with stale configuration ([#2595](https://github.com/defenseunicorns/uds-core/pull/2595))
- **Optional disabling of inactive users:** UDS Identity Config 0.26.0 adds the `disable-inactive-users` Keycloak workflow for automatically disabling non-admin accounts after a configurable inactivity window. The workflow is disabled by default and only activates when `ACCOUNT_INACTIVITY_DAYS` is set during initial realm import. See [Configure automatic account inactivity disable](/how-to-guides/identity-and-authorization/configure-account-inactivity-disable/) for configuration and verification steps.

## Upgrade considerations

### Identity Config updates (0.26.1)

UDS Core 1.3.0 includes UDS Identity Config [0.26.1](https://github.com/defenseunicorns/uds-identity-config/releases/tag/v0.26.1), which was also included in the 1.2.2 patch release. No breaking changes or manual realm steps are required.
