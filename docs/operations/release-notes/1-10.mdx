### ⚠ Breaking changes

| Change | Impact | Action required |
|---|---|---|
| UDS Core no longer enables Keycloak's generic `preview` feature group by default. | Keycloak preview features that were previously activated indirectly, including `stateless:v1` in Keycloak 26.7, no longer start unless you configure them. | Review deployments that rely on preview features. Add the specific feature names to `keycloak.keycloak.features.additional`, or add `preview` only when you intentionally require every preview feature. |

### Notable features

- **Explicit Keycloak feature selection:** UDS Core always enables the required `fips` feature and exposes the `keycloak.keycloak.features.additional` value for additional features. UDS Core no longer passes `preview` by default, which prevents newly added Keycloak preview features from activating during a UDS Core upgrade. See the [identity and authorization configuration reference](/reference/configuration/identity-and-authorization/#keycloak-features).

### Dependency updates

The current feature change does not bump a runtime dependency. Add dependency updates from the full 1.10 commit range when the release changelog is generated.

## Upgrade considerations

### Pre-upgrade steps

<Steps>

1. **Review Keycloak feature dependencies**

   If your deployment relies on a Keycloak feature that UDS Core previously enabled through the generic `preview` group, add that feature to `keycloak.keycloak.features.additional` before upgrading. Prefer the specific feature name over `preview` so unrelated preview features do not activate.

</Steps>

### Post-upgrade verification

<Steps>

1. **Verify Keycloak startup and SSO flows**

   Confirm that the Keycloak StatefulSet becomes ready and test the SSO flows used by your deployment. If you enabled an additional feature, verify its expected behavior separately.

</Steps>
