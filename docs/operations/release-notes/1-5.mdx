> [!NOTE]
> For general upgrade procedures, see the [Upgrade Overview](/operations/upgrades/overview/).

UDS Core 1.5 migrates the unicorn flavor from RapidFort to Chainguard FIPS hardened images, introduces UDS Portal as a new optional functional layer in the upstream flavor, and stabilizes Keycloak on smaller nodes by disabling Infinispan virtual threads. Operators on the unicorn flavor must allow outbound access to `cgr.dev/defenseunicorns.com`, and RHEL 9 hosts running fapolicyd in enforcing mode need a rule to permit CNI execution from `/opt/cni/bin/.cgr/`.

### ⚠ Breaking changes

| Change                                                              | Impact                                                                                                                                                                                                              | Action required                                                                                                                                                                                                                                                                                                          |
| ------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Unicorn flavor images move from RapidFort to Chainguard FIPS images | All unicorn images are now pulled from `cgr.dev/defenseunicorns.com/*-fips` instead of `quay.io/rfcurated/*`. Clusters on the unicorn flavor need outbound network access to `cgr.dev/defenseunicorns.com`.         | Allow outbound network access to `cgr.dev/defenseunicorns.com` from cluster nodes. On RHEL 9 with fapolicyd in enforcing mode, add a rule to allow execution from `/opt/cni/bin/.cgr/` because the Chainguard CNI image writes `libcrypto.so.3` into that path at runtime. The RKE2 user data script handles this automatically. |

### Notable features

- **UDS Portal as a new functional layer:** adds `packages/portal/` and `src/portal/` deploying [uds-portal](https://github.com/defenseunicorns/uds-portal) v0.2.0 as an optional layer. Available in the upstream flavor only. Follows the same `DOMAIN`/`ADMIN_DOMAIN` variable pattern as other layers ([#2699](https://github.com/defenseunicorns/uds-core/pull/2699)).
- **Unicorn flavor: Chainguard FIPS images:** the unicorn flavor migrates from RapidFort to Chainguard FIPS hardened images. Chainguard images are built on distroless bases and satisfy FIPS 140-2 requirements. The Pepr `validateIstioImage` policy is updated to recognize `cgr.dev/defenseunicorns.com/istio-proxy-fips` as the valid unicorn proxy image. The Keycloak unicorn image requires an explicit JCE security provider override, which UDS Core injects internally via a ConfigMap and `JAVA_OPTS_APPEND`, so no operator action is required ([#2650](https://github.com/defenseunicorns/uds-core/pull/2650)).
- **Keycloak: disable Infinispan virtual threads:** Infinispan 16 (shipped with Keycloak 26.6) enables virtual threads by default, which can deadlock under Kubernetes thread-pool starvation on nodes with fewer than 4 vCPUs. UDS Core now sets `-Dorg.infinispan.threads.virtual=false` to prevent this. References: [keycloak#48792](https://github.com/keycloak/keycloak/issues/48792), [keycloak#49203](https://github.com/keycloak/keycloak/issues/49203) ([#2686](https://github.com/defenseunicorns/uds-core/pull/2686)).

## Upgrade considerations

The unicorn flavor change from RapidFort to Chainguard constitutes a breaking change but requires no steps beyond the standard upgrade procedure, except for fapolicyd users on RHEL 9 (see the Breaking changes table above).
