UDS Core 1.2 adds signed JWT client authentication for the UDS Operator's Keycloak communication, upstream Istio network policies for all Istio components, and automatic user account disable after configurable inactivity. This release also brings Keycloak waypoint pod annotation and label overrides for easier integration with service mesh tooling.

### Notable features

- **Signed JWT operator authentication:** the UDS Operator now supports signed JWT client authentication for Keycloak API calls, with an automatic fallback to client secret. The default mode is `CLIENT_SECRET`; opt in via the `KEYCLOAK_CLIENT_MODE` operator config value set to `SIGNED_JWT` or `AUTO` ([#2580](https://github.com/defenseunicorns/uds-core/pull/2580))
- **Upstream Istio network policies:** enables the upstream Istio Helm chart NetworkPolicies for all Istio components (istiod, istio-cni, ztunnel, and all gateways), restricting ingress to known ports while allowing all egress ([#2564](https://github.com/defenseunicorns/uds-core/pull/2564))
- **User account inactivity disable:** adds a `realmConfig.accountInactivityDays` setting that automatically disables non-admin user accounts after a configurable period of inactivity. Set to `0` to disable this feature ([#2566](https://github.com/defenseunicorns/uds-core/pull/2566))
- **Keycloak waypoint annotation/label overrides:** adds `waypoint.deployment.podAnnotations` and `waypoint.deployment.podLabels` Helm values for the Keycloak waypoint pod ([#2565](https://github.com/defenseunicorns/uds-core/pull/2565))
