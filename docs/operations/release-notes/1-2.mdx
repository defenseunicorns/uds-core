UDS Core 1.2 adds upstream Istio network policies for all Istio components and also brings Keycloak waypoint pod annotation and label overrides for easier integration with service mesh tooling.

### Notable features

- **Upstream Istio network policies:** enables the upstream Istio Helm chart NetworkPolicies for all Istio components (istiod, istio-cni, ztunnel, and all gateways), restricting ingress to known ports while allowing all egress ([#2564](https://github.com/defenseunicorns/uds-core/pull/2564))
- **Keycloak waypoint annotation/label overrides:** adds `waypoint.deployment.podAnnotations` and `waypoint.deployment.podLabels` Helm values for the Keycloak waypoint pod ([#2565](https://github.com/defenseunicorns/uds-core/pull/2565))

## Upgrade considerations

> [!IMPORTANT]
> Skip v1.2.0 and upgrade directly to v1.2.1. The v1.2.0 release inherited an upstream Helm chart change that renamed the Loki write service account, breaking deployments that reference the `loki` service account by name for object storage auth (e.g. IRSA). v1.2.1 restores the shared `loki` service account for write pods ([#2599](https://github.com/defenseunicorns/uds-core/pull/2599), upstream [grafana-community/helm-charts#390](https://github.com/grafana-community/helm-charts/issues/390)).
